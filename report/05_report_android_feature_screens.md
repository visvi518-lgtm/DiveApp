# Phase 5 Report — Android Feature Screens

## 배경 / 번호 정리

`Report/04_report_android_foundation.md`에서 이미 "Phase 4"라는 이름을 Android 기반 구축에 사용해버려 번호가 꼬였다. 원래 계획(맨 처음 세운 5단계)의 "Phase 4 = 기능 화면 구현"에 해당하는 작업을 이번 리포트에서 다루며, 파일 번호는 05로 이어간다. 앞으로는 "Phase" 대신 문서 번호로 지칭하는 편이 안전하다.

이번 작업은 사용자가 "Android 기능 화면 구현"을 선택해 진행했다.

---

## 요약

`07_Screens.md` 기준 5개 도메인(Dive Log, CO₂ Table, Certification, Community, Information)의 화면을 백엔드 Phase 2 API와 실제로 연동되는 형태로 구현했다. 탭 구조를 각 탭이 자신만의 중첩 네비게이션(list → detail → create/edit)을 갖도록 재구성했고, Information은 홈 탭에, Certification은 마이페이지 탭에 중첩 화면으로 연결했다.

**이번에도 Android는 실제로 빌드하고, 에뮬레이터에 설치하고, 화면을 하나하나 탭하며 렌더링과 동작을 확인했다.**

---

## 구현 범위

### 1. Certification (마이페이지 → 자격증 관리)
- 목록/등록/상세/수정 화면, 발급 기관 드롭다운, 발급일·만료일 Material3 DatePicker
- 백엔드가 부분 수정(partial update)을 지원하지만, 앱은 항상 폼 전체를 채워서 보내는 방식으로 구현 — kotlinx.serialization이 모든 필드를 인코딩하기 때문에 부분 필드만 보내면 의도치 않게 나머지 필드를 null로 덮어쓸 위험이 있어, 필드 값을 모두 채워 보내는 방식으로 그 위험을 원천 차단했다.

### 2. Information (홈 → 정보 게시판)
- 목록(썸네일 카드)/상세 화면. 읽기 전용, 인증 불필요.

### 3. CO₂ Table (Training)
- 프로그램 설정(세트 수 슬라이더 5~20, 휴식/숨참기 시간, 세트당 변화량) → 실행(휴식→숨참기 반복, 실시간 카운트다운, 일시정지/종료) → 결과(완료 세트 수, 저장) → 기록(통계 + 이력 리스트)
- 타이머는 전적으로 클라이언트에서 코루틴으로 동작하며(`Docs/02_Requirements.md` 3.3과 일치), 서버에는 결과만 저장

### 4. Dive Log
- 목록(통계 카드 + 로그 리스트) / 작성(종류 선택 → 위치 → 종목별 상세) / 상세 / 수정
- 백엔드가 수정 시 위치·종류 변경을 지원하지 않는 것에 맞춰, 수정 화면은 메모와 종목별 수치만 변경 가능하도록 별도로 단순화해서 구현 (작성 화면과 의도적으로 다른 폼)
- Naver Map API 연동 없이 위도/경도를 직접 입력하는 방식으로 임시 구현 (아래 백로그 참고)

### 5. Community
- 게시판(단일) → 게시글 목록 → 상세(댓글 포함) → 글쓰기/수정
- 댓글 작성/삭제, 게시글 작성자 여부에 따라 수정/삭제 버튼 노출 여부를 클라이언트에서도 판단 (서버가 최종 권한 검증은 별도로 수행)

### 6. 공통
- `DateSerializers.kt`: 백엔드의 DATE/TIMESTAMPTZ 컬럼을 `java.time.LocalDate`/`OffsetDateTime`으로 변환하는 kotlinx.serialization 커스텀 시리얼라이저 추가
- `AppContainer`에 5개 도메인의 Service/Repository 모두 등록
- 탭 구조를 각자 독립된 `NavHost`를 갖도록 재구성 (Home/DiveLog/Training/Community/MyPage)

## 실제 빌드·실행 검증 (중요)

이전 리포트와 마찬가지로 실제 Gradle 빌드 + 에뮬레이터 실행까지 확인했다.

- 최초 빌드에서 컴파일 에러 **단 1건** 발생(`ExposedDropdownMenu`를 잘못 import) — 즉시 수정 후 클린 빌드 성공. 그 외 deprecated 아이콘/API 경고 3건도 모두 정리해 경고 0건으로 마무리.
- 19.2MB APK를 Pixel_9 에뮬레이터에 설치, 실행 확인.
- **백엔드 서버가 이 세션에 떠 있지 않아** 로그인 이후 화면을 정상 경로로는 확인할 수 없었다. 이를 우회하기 위해 `AuthSession`의 초기 상태를 일시적으로 `AUTHENTICATED`로 하드코딩하여 빌드 → 5개 탭 전체와 하위 화면(다이브 로그 작성 폼, CO₂ Table 타이머 실행 화면, 자격증 관리 중첩 네비게이션 등)을 직접 탭하며 스크린샷으로 확인 → **작업 완료 즉시 하드코딩을 원복하고 다시 빌드해 정상 로그인 화면으로 돌아오는 것까지 재확인**했다.
  - 확인된 것: 홈(배너/정보게시판 카드), 다이브 로그 목록(에러 상태) 및 작성 폼(라디오 버튼, 날짜 선택, 종목별 필드) 전체 렌더링, CO₂ Table 설정 화면과 **실제로 2:00→1:57로 초 단위 카운트다운되는 실행 화면**, 커뮤니티 목록(에러 상태), 마이페이지 및 자격증 관리 중첩 네비게이션.
  - 모든 화면에서 크래시(FATAL, AndroidRuntime 예외) 없음을 logcat으로 확인.
  - 백엔드가 없어 발생하는 "네트워크 연결을 확인해주세요" 에러 상태 자체도 각 목록 화면에서 정상적으로 표시됨을 확인 (에러 처리 경로 검증).
