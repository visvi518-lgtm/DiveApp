# Workflow

## Standard Development Workflow

Every task must follow this workflow.

```
Receive Task
    ↓
Read CLAUDE.md
    ↓
Read Related Documents
    ↓
Analyze Current Project State
    ↓
Requirement Analysis
    ↓
Architecture Review
    ↓
UI Design (if needed)
    ↓
Database Design (if needed)
    ↓
API Design (if needed)
    ↓
Implementation Plan
    ↓
User Approval (required for architecture or database changes)
    ↓
Implementation
    ↓
Self Review
    ↓
Testing
    ↓
Update Documentation
    ↓
Git Commit
    ↓
Complete
```

---

## Mandatory Rules

- Never skip any step.
- Never implement before understanding the project.
- Never change architecture without approval.
- Never create unnecessary files.
- Always update documentation after implementation.
- Always review your own code before finishing.

---

## Definition of Done

A task is complete only if all of the following are true.

- Requirements satisfied
- Code implemented
- Tests completed
- Documentation updated
- No critical warnings
- Project structure maintained
